# shadcn/ui Monochrome Redesign — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Migrate the admin area and the editor chrome from ad-hoc CSS modules + inline `style={{}}` to a single shadcn/ui design system — monochrome, light/dark switchable, native animations — without touching the landing page.

**Architecture:** Tailwind v4 + shadcn/ui installed globally, but Tailwind's base reset (preflight) is replaced by a scoped reset under a `[data-adv-ui]` wrapper that only the admin layout and editor-chrome roots carry. The landing page never has `[data-adv-ui]`, so it is byte-for-byte unchanged. Theme tokens are monochrome CSS variables scoped to `[data-adv-ui]`; `next-themes` toggles a `.dark` class on `<html>`.

**Tech Stack:** Next.js 16.2.6 (App Router, Turbopack), React 19.2.4, TypeScript 5, Tailwind CSS v4, shadcn/ui, next-themes, lucide-react, sonner.

**Note on testing:** This is a styling/design-system migration — no new business logic, so there are no new unit tests to write. The verification for every task is: `npx tsc --noEmit` clean, the existing 83 tests still green (`npm test`), `npm run build` succeeds, and — where the task changes rendered UI — a manual browser check in both light and dark mode plus a confirmation the landing page is visually unchanged. The local admin password is `localdev123` (set in `.env.local` this session); log in at `http://localhost:3000/admin/login`, the editor is at `http://localhost:3000/` while logged in.

**Commits:** Commit after every task. Push is left to the user.

```tsx
"use client";
import { useTheme } from "next-themes";
import { useEffect, useState } from "react";
import { Moon, Sun } from "lucide-react";
import { Button } from "@/components/ui/button";

export function ThemeToggle() {
  const { resolvedTheme, setTheme } = useTheme();
  const [mounted, setMounted] = useState(false);
  useEffect(() => setMounted(true), []);

  return (
    <Button
      variant="ghost"
      size="icon"
      aria-label="Toggle theme"
      onClick={() => setTheme(resolvedTheme === "dark" ? "light" : "dark")}
    >
      {mounted && resolvedTheme === "dark" ? <Sun className="size-4" /> : <Moon className="size-4" />}
    </Button>
  );
}
```

```tsx
"use client";
import type { ReactNode } from "react";
import {
  AlertDialog, AlertDialogAction, AlertDialogCancel, AlertDialogContent,
  AlertDialogDescription, AlertDialogFooter, AlertDialogHeader, AlertDialogTitle,
} from "@/components/ui/alert-dialog";
import { buttonVariants } from "@/components/ui/button";
import { cn } from "@/lib/utils";

export function ConfirmDialog({
  open, title, description, confirmLabel, cancelLabel, destructive = false, onConfirm, onCancel,
}: {
  open: boolean;
  title: string;
  description?: ReactNode;
  confirmLabel: string;
  cancelLabel: string;
  destructive?: boolean;
  onConfirm: () => void;
  onCancel: () => void;
}) {
  return (
    <AlertDialog open={open} onOpenChange={(o) => { if (!o) onCancel(); }}>
      <AlertDialogContent>
        <AlertDialogHeader>
          <AlertDialogTitle>{title}</AlertDialogTitle>
          {description && <AlertDialogDescription asChild><div>{description}</div></AlertDialogDescription>}
        </AlertDialogHeader>
        <AlertDialogFooter>
          <AlertDialogCancel onClick={onCancel}>{cancelLabel}</AlertDialogCancel>
          <AlertDialogAction
            onClick={onConfirm}
            className={cn(destructive && buttonVariants({ variant: "destructive" }))}
          >
            {confirmLabel}
          </AlertDialogAction>
        </AlertDialogFooter>
      </AlertDialogContent>
    </AlertDialog>
  );
}
```
